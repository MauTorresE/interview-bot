# BLOCKING CONSTRAINTS â€” Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure

- [ ] CONSTRAINT: `session.say()` from the timing loop â€” Calling `session.say()` at 80/90/100% causes voice overlap with LLM TTS output. Structural mitigation: all timing tiers below 110% ONLY update state prompts via `_update_instructions()` or force closing via `_force_llm_closing()`. The Tier 0 rewrite removed all `session.say()` calls from timing tiers entirely. Do NOT reintroduce them.

- [ ] CONSTRAINT: Event handlers registered after `session.start()` â€” LiveKit Agents 1.5.2 silently loses events registered after `await session.start(agent=agent, room=ctx.room)`. Structural mitigation: register ALL `@session.on(...)` handlers BEFORE the `await session.start(...)` call. This includes the new `agent_state_changed` handler added in Wave 1.3.

- [ ] CONSTRAINT: Fixed USD pricing for MX market â€” $5,000 USD (~$95K MXN) anchors way too high for Mexican small businesses. Mitigation: use MXN ranges ($15K-$35K per phase) and label as "estimados iniciales". The `/generate-proposal` skill enforces this. Do NOT introduce USD pricing in any user-facing copy.

- [ ] CONSTRAINT: `session.room` access after `aclose()` â€” `_send_data` crashes with AttributeError if called after the session closes. Structural mitigation: the helper already checks `hasattr(self.session, 'room') and self.session.room`. Also: always send data channel messages BEFORE any `session.aclose()`. The Tier 0 `_complete_interview` helper in `on_data` follows this pattern.

- [ ] CONSTRAINT: Proposing during discovery call â€” Agent acting as a consultant that proposes solutions during the call breaks NSS model. Structural mitigation: system prompt in `agent/interview_prompts.py` explicitly frames the agent as an info-gatherer only. Opportunities are captured post-call via the `/generate-proposal` skill. Do NOT edit the prompt to encourage proposing.

- [ ] CONSTRAINT: Anthropic prompt cache vs `update_instructions()` â€” mid-session instruction updates are UNRELIABLE when `caching="ephemeral"` is set. The cached prefix keeps serving stale system prompts for several turns. Structural mitigation: use `session.generate_reply(instructions="...")` for any behavior change that MUST take effect immediately. Wave 1.1's `_force_llm_closing()` is the canonical example.

- [ ] CONSTRAINT: Deepgram interim segments have rotating IDs â€” Interim transcription segments often have new `seg.id` values AND cumulative text ("A" â†’ "A B" â†’ "A B C"). Naive append produces "A A B A B C" duplication. Structural mitigation: Wave 1.6's per-speaker `liveInterim` slot pattern â€” interim updates REPLACE the slot, finals commit to entries. Do NOT revert to append-on-interim logic.

- [ ] CONSTRAINT: LobbyScreen must NOT use LiveKitRoom â€” Using LiveKitRoom in the lobby grabs the mic and doesn't release it cleanly before the interview room mounts. Structural mitigation: lobby uses native `getUserMedia` + Web Audio API + 1000ms release delay before transitioning. Do NOT refactor lobby-screen.tsx to use LiveKit hooks.
