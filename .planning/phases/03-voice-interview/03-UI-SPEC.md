### Phase 3 Accent Usage (explicit list)

Accent (`--primary` / #6366F1) is reserved for:
- Orb base color (all states) -- the morphing violet orb IS the accent
- Orb glow effects (box-shadow with primary color at varying opacities)
- "Comenzar" CTA button in lobby (filled background)
- Active mic level meter bar fill
- Focus rings on text input and controls
- Phase indicator text color
- "Ver transcripcion" link in respondents tab

Accent is NOT used for: transcript text, speaker labels, timer text, orb state labels, mic mute button.

## Interaction Patterns

### Page Phase Transitions (D-28)

| Transition | Trigger | Animation |
|------------|---------|-----------|
| Consent -> Lobby | Consent form "Comenzar" click + API success | 300ms opacity fade-out on consent, 300ms opacity fade-in on lobby |
| Lobby -> Interview | Lobby "Comenzar entrevista" click + LiveKit connection established | 300ms opacity fade-out on lobby, 300ms opacity fade-in on interview room |
| Interview -> Completion | Agent sends end signal OR respondent clicks "Terminar" | Orb: 300ms opacity fade-out. Card: 300ms delay, then 300ms translateY(20px->0) + opacity fade-in |

### Orb Animation States (D-29, D-30)

| State | Detection | Visual |
|-------|-----------|--------|
| Idle/Connecting | `useConnectionState() !== Connected` OR no agent activity | Static circle, subtle glow, gentle scale pulse: `scale(1.0) -> scale(1.03)`, 2s `ease-in-out` infinite loop |
| Listening | `Connected` AND NOT `useIsSpeaking(agent)` AND no thinking signal | Breathe animation (scale 1.0->1.05->1.0, 3s loop), border glow intensifies, ring ripple outward every 1.5s (concentric circle expanding from orb, fading to 0 opacity at 2x orb radius) |
| Thinking | Agent data channel message `{"type": "thinking"}` received | Orb contracts to `scale(0.95)`, internal shimmer effect: rotating radial-gradient (violet-to-transparent sweep, 400ms rotation cycle). Orb opacity 0.85. |
| Speaking | `useIsSpeaking(agent) === true` | `border-radius` morphing between organic blob shapes keyed to audio amplitude from `useTrackVolume(agentTrack)`. Amplitude 0-1 maps to border-radius variance 50% +/- 15%. Active, undulating surface feel. Glow at maximum intensity. |

State transitions: 250-300ms `ease-out` on all properties (scale, opacity, box-shadow, border-radius).

Implementation approach:
- CSS `@keyframes` for idle pulse, listening breathe, listening ripple, thinking shimmer
- Web Audio API `AnalyserNode` for amplitude-reactive border-radius morphing during speaking state
- CSS `filter: blur()` for glow effects
- No WebGL or canvas required

### Mic Level Meter (Lobby)

| State | Visual |
|-------|--------|
| No audio | Empty bar, `--border` background, "No se detecta audio..." in `--destructive` text |
| Audio detected | Green fill (#22c55e) proportional to volume level (0-100%), "Audio detectado" in success green text |
| Mic permission denied | Empty bar, `--destructive` background pulse, mic permission denied error copy |

Meter updates driven by `useTrackVolume()` at native refresh rate (typically 60fps). Fill bar uses CSS `width` transition of 50ms for smooth visual.

### Text Fallback Input (D-03)

| State | Visual |
|-------|--------|
| Empty | Placeholder text "Escribe tu respuesta...", `--muted-foreground` |
| Typing | `--foreground` text, `--primary` focus ring |
| Submit | Enter key or send button submits via data channel (`"text-input"` topic), input clears |
| Disabled | During agent speaking state (prevents overlapping), opacity 0.5 |

Send button: lucide `SendHorizontal` icon, 20px, `--primary` color, inside input bar right side.

### End Interview (D-04)

| State | Visual |
|-------|--------|

| Button hover | `--destructive` at 10% opacity background |
| Confirmation | Alert dialog: heading "Terminar entrevista?", body copy, "Cancelar" + "Terminar" (destructive filled) buttons |
| After confirm | Agent receives end signal, performs closing summary, then triggers completion card |

### Mic Mute Toggle

| State | Visual |
|-------|--------|
| Unmuted (default) | lucide `Mic` icon, 20px, `--foreground`, 44x44px button, `--card` background |

### Connection States

| State | Visual |
|-------|--------|
| Connecting | Orb in idle state, lobby shows "Conectando..." |
| Connected | Orb transitions to listening state |
| Reconnecting | Sonner toast: "Se perdio la conexion. Intentando reconectar..." with amber indicator. Orb returns to idle state. |
| Disconnected (failed) | Sonner toast: "No se pudo reconectar." persistent. Interview room dims to 50% opacity. |

### Transcript Auto-Scroll

- New entries trigger smooth scroll to bottom (`scrollIntoView({ behavior: 'smooth' })`)
- If user scrolls up (scroll position > 100px from bottom), auto-scroll pauses
- "New message" indicator appears at bottom when paused and new entries arrive
- Clicking the indicator resumes auto-scroll

## Accessibility Notes

- Interview room controls must have `aria-label` attributes (mic toggle, end interview, send text)
- Orb state label serves as `aria-live="polite"` region so screen readers announce state changes
- Transcript feed is an `aria-live="polite"` region with `role="log"`
- Text fallback input ensures keyboard-only users can participate without voice
- Phase indicator uses `aria-live="polite"` for screen reader announcements
- Focus management: after phase transitions, focus moves to the primary interactive element (lobby: mic selector, interview: text input, completion: none)
- Color contrast: all text meets WCAG AA on dark backgrounds (verified by Phase 1 palette)
