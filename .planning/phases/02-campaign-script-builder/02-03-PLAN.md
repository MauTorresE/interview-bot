**1. `src/app/(dashboard)/campaigns/[id]/actions.ts`** — All campaign detail server actions.

Mark `'use server'` at top. Import patterns from settings/actions.ts.

Define these exported async functions:

**`saveBrief(campaignId: string, input: ResearchBriefInput): Promise<ActionResult>`**
- Validate with `researchBriefSchema.safeParse(input)`, error: "Datos de guia invalidos."
- Auth check: getUser, get orgId
- Admin client upsert into `research_briefs`: `{ campaign_id: campaignId, org_id: orgId, brief_data: parsed.data }` with onConflict `campaign_id`.
- `revalidatePath(\`/campaigns/${campaignId}\`)`
- Return `{ success: true }`
- Error: "No se pudo guardar la guia. Intenta de nuevo."

**`addRespondent(campaignId: string, input: AddRespondentInput): Promise<ActionResult>`**
- Validate with `addRespondentSchema.safeParse(input)`, error: "Datos del participante invalidos."
- Auth check: getUser, get orgId
- Admin client insert into `respondents`: `{ campaign_id: campaignId, org_id: orgId, name, email, notes: notes ?? null }`
- invite_token is auto-generated by DB default (gen_random_uuid())
- `revalidatePath(\`/campaigns/${campaignId}\`)`
- Return `{ success: true }`
- Error: "No se pudo agregar al participante. Verifica el email e intenta de nuevo."

**`deleteRespondent(campaignId: string, respondentId: string): Promise<ActionResult>`**
- Auth check: getUser, get orgId
- Admin client delete from `respondents` where id = respondentId AND org_id = orgId
- `revalidatePath(\`/campaigns/${campaignId}\`)`
- Return `{ success: true }`
- Error: "No se pudo eliminar al participante. Intenta de nuevo."

**`sendReminder(campaignId: string, respondentId: string): Promise<ActionResult>`**
- Auth check: getUser
- STUBBED: `console.log(\`[STUB] Reminder sent for respondent ${respondentId} in campaign ${campaignId}\`)`
- Return `{ success: true }`
- Note in comment: "// TODO: Wire up email service (Resend/Supabase Edge Functions) in future phase"

**`archiveCampaign(campaignId: string): Promise<ActionResult>`**
- Auth check: getUser, get orgId
- Admin client update campaigns set `status = 'archived'` where id = campaignId AND org_id = orgId
- `revalidatePath('/campaigns')`
- Return `{ success: true }`
- Error: "No se pudo archivar la campana. Intenta de nuevo."

**`updateCampaignConfig(campaignId: string, input: Partial<{ name: string; description: string; language: string; duration_target_minutes: number; voice_provider: string; voice_id: string; interviewer_style: string; reusable_invite_enabled: boolean }>): Promise<ActionResult>`**
- Auth check: getUser, get orgId
- Admin client update campaigns with only the provided fields
- `revalidatePath(\`/campaigns/${campaignId}\`)`
- Return `{ success: true }`
- Error: "No se pudo actualizar la configuracion. Intenta de nuevo."

| Client -> Server Action | Brief content, respondent data crosses from browser to server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | saveBrief server action | mitigate | Zod validation via researchBriefSchema; max lengths prevent oversized content; org_id from JWT |
| T-02-07 | Tampering | addRespondent server action | mitigate | Zod validation via addRespondentSchema; email format enforced; org_id from JWT |
| T-02-08 | Information Disclosure | Respondent invite tokens | accept | UUIDs are 128-bit random via gen_random_uuid(); infeasible to brute-force. Token only grants access to consent page, not data |
| T-02-09 | Tampering | deleteRespondent server action | mitigate | Admin client delete includes org_id = orgId condition; cannot delete respondents in other orgs |
| T-02-10 | Spoofing | XSS in brief content | mitigate | React auto-escapes all JSX; brief content rendered as text nodes, never dangerouslySetInnerHTML |
</threat_model>
