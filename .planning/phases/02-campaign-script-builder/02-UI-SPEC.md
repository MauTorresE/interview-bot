## Layout Contracts

### Campaign List Page (`/campaigns`)

**Decision D-02 resolution:** Hybrid layout — card grid for visual scanning with table-density information.

```
+----------------------------------------------------------+
| Sidebar | Campanas                          [+ Crear]    |
|         |                                                |
|         | [Filter: Status v]  [Search...]                |
|         |                                                |
|         | +--Card--+ +--Card--+ +--Card--+               |
|         | | Status | | Status | | Status |               |
|         | | Name   | | Name   | | Name   |               |
|         | | Desc   | | Desc   | | Desc   |               |
|         | | 3/10   | | 7/7    | | 0/5    |               |
|         | | Lang   | | Lang   | | Lang   |               |
|         | +--------+ +--------+ +--------+               |
|         |                                                |
+----------------------------------------------------------+
```

- Layout: CSS Grid, 3 columns on desktop (>=1024px), 2 on tablet (768-1023px), 1 on mobile (<768px)
- Card dimensions: min-width 280px, flexible height
- Card contents (top to bottom): status badge (top-right), campaign name (semibold 14px), description (truncated to 2 lines, 14px muted), progress bar with "N/M entrevistas" label below, language tag (12px label), created date (12px muted)
- Card hover: border transitions from `--border` to `--primary` at 30% opacity, 150ms ease
- Card click: navigates to `/campaigns/[id]`
- Top bar: page heading "Campanas" (20px heading), "Crear campana" button (accent filled, top-right)
- Filter row: status dropdown filter + text search input, 16px below heading, 16px above grid
- Page padding: 32px horizontal, 32px vertical (desktop); 16px (mobile)

### Campaign Creation Dialog

**Decision D-01 resolution:** Dialog with minimal fields. Progressive disclosure — full configuration happens on the campaign detail page after creation (per D-14).

```
+--Dialog (max-width 480px)----+
| Crear campana            [X] |
|                              |
| Nombre *                     |
| [________________________]   |
|                              |
| Descripcion                  |
| [________________________]   |
| [________________________]   |
|                              |
| Idioma *         Duracion *  |
| [Espanol v]     [15 min v]   |
|                              |
| [Cancelar]    [Crear campana]|
+------------------------------+
```

- Trigger: "Crear campana" button on campaign list page
- Fields: name (required, Input), description (optional, Textarea, 3 rows), language (required, Select, default "Espanol"), duration (required, Select, options: 10/15/30 min, default 15)
- Dialog width: max-width 480px
- Padding: 24px
- Form gap: 24px between fields
- Two-column row for language + duration (each 50% width, 16px gap)
- Footer: "Cancelar" (ghost button) left, "Crear campana" (accent filled) right
- On success: close dialog, redirect to new campaign detail page, sonner toast "Campana creada"

|         |                                                |
|         |  (Tab content area — see each tab below)       |

- Breadcrumb: "Campanas" (link) > "Campaign Name" (current)
- Actions dropdown (top-right): "Editar detalles", "Duplicar campana", "Archivar campana" (destructive)
- Tabs: Resumen, Guia de investigacion, Participantes, Configuracion
- Active tab: accent underline (3px bottom border, `--primary`), text `--primary-text`
- Inactive tab: no underline, text `--secondary-text`, hover `--primary-text`
- Tab content padding: 32px top from tab bar

#### Tab: Guia de Investigacion (Research Brief)

**Decision D-06 resolution:** Hybrid approach — structured form with labeled sections, each containing a Textarea for freeform input plus optional helper chips/prompts.

**Decision D-08 resolution:** CONF-01/CONF-02 reinterpreted. No branching UI. Instead, "critical paths" section lets researcher define conditional exploration goals.

```
+--Content Area----------------------------------------+
|                                                       |
|  Guia de investigacion                    [Guardar]  |
|                                                       |
|  +--Section 1: Objetivos de investigacion-----------+ |
|  | Que quieres aprender de esta investigacion?      | |
|  | [Textarea — 4 rows]                              | |
|  +--------------------------------------------------+ |
|                                                       |
|  +--Section 2: Datos criticos-----------------------+ |
|  | Informacion especifica que el AI debe obtener    | |
|  | [Textarea — 4 rows]                              | |
|  |                                                  | |
|  | Caminos criticos (opcional)                      | |
|  | Si el participante menciona X, explorar Y        | |
|  | [+ Agregar camino]                               | |
|  | +-- Path 1: Si [___] entonces explorar [___] --+ | |
|  | +-- Path 2: Si [___] entonces explorar [___] --+ | |
|  +--------------------------------------------------+ |
|                                                       |
|  +--Section 3: Contexto-----------------------------+ |
|  | Que debe saber el AI sobre el tema/empresa?      | |
|  | [Textarea — 4 rows]                              | |
|  +--------------------------------------------------+ |
|                                                       |
|  +--Section 4: Tono y enfoque-----------------------+ |
|  | Como debe manejar la conversacion el AI?         | |
|  | [Textarea — 3 rows]                              | |
|  +--------------------------------------------------+ |
|                                                       |
|  [Vista previa]                                      |
+-------------------------------------------------------+
```

- Max content width: 720px, centered
- Each section: Card with 24px padding, 16px gap between sections
- Section heading: 14px / 600 weight
- Section helper text: 12px / `--secondary-text`, below heading, above textarea
- Textarea: `--surface-2` background, `--border` border, 14px body text, min-height per section (see above)
- Critical paths: repeatable row pattern — "Si" + Input (flex-1) + "entonces explorar" + Input (flex-1) + delete icon button (ghost, lucide X, 16px). Add button: ghost button with lucide Plus icon, 14px text
- "Guardar" button: accent filled, top-right of content area, sticky at top on scroll
- "Vista previa" button: ghost button, bottom of form, opens preview dialog

**Decision D-07 resolution:** Script preview is a read-only dialog that renders the brief as the AI would see it — a formatted system prompt summary. Not a simulated interaction (too complex for Phase 2, simulated chat is Phase 3 territory).

Preview dialog:
- Max-width 640px
- Title: "Vista previa de la guia"
- Body: formatted read-only rendering of the 4 sections with section headings, content, and critical paths as bullet points
- Monospace font (Geist Mono) for the "system prompt" representation
- Footer: "Cerrar" ghost button only

- Actions per row (dropdown menu): "Enviar recordatorio", "Copiar enlace", "Eliminar" (destructive)
- "Agregar" button: opens dialog to add single respondent (name, email, notes)

- Table row hover: `--surface-3` background
- Respondent count shown in tab label: "Participantes (N)"

**Decision D-09 resolution:** Both unique and reusable links. Each respondent gets a unique link (auto-generated on add). Campaign also has one reusable link (toggle on/off in config tab). Reusable link accessible from "Enlace" button.

Add respondent dialog:
- Max-width 480px
- Fields: Nombre (required), Email (required), Notas (optional, Textarea 2 rows)
- Footer: "Cancelar" ghost, "Agregar participante" accent filled
- On success: close dialog, respondent appears in table, sonner toast "Participante agregado"

**Decision D-12 resolution:** Voice persona as a list with play preview buttons. Not a grid (too few voices initially to justify grid) and not a bare dropdown (voice requires audio preview).

Voice persona list:
- Each row: radio selection + voice name + provider label (12px muted) + play button (lucide Play, ghost icon button, 36px touch target)
- Play button plays a 3-5 second sample clip
- Currently playing: Play icon becomes Pause, accent color
- Premium voices: "Premium" badge (12px, muted border) after provider name
- Selected voice: accent left border (3px), tinted background (primary 10% opacity) — same pattern as sidebar active item

**Decision D-13 resolution:** Toggle group (4 options). Each option is a pressable segment showing the style name.

Interviewer style toggle group:
- ToggleGroup with 4 items: Profesional, Casual, Empatico, Directo
- Selected: accent background, white text
- Unselected: `--surface-2` background, `--secondary-text`
- Below toggle group: description text (14px, `--secondary-text`) that changes based on selection

  - Casual: "Tono conversacional y relajado. Ideal para estudios de consumidor."
  - Empatico: "Tono calido y comprensivo. Ideal para temas sensibles."
  - Directo: "Tono conciso y enfocado. Ideal para entrevistas cortas."

## Interaction Patterns

### Campaign List

| State | Visual Treatment |
|-------|-----------------|
| Loading | 6 skeleton cards in grid layout (matches card dimensions) |
| Empty | Centered empty state (lucide FolderOpen 48px muted, heading, body, accent CTA) |
| Populated | Card grid, sorted by last updated (most recent first) |
| Card hover | Border color transitions to `--primary` at 30% opacity, 150ms ease |
| Filter active | Status dropdown shows selected value, filtered cards animate with opacity 150ms |

### Campaign Creation Dialog

| State | Visual Treatment |
|-------|-----------------|
| Dialog open | Overlay backdrop `hsl(0 0% 0% / 0.8)`, dialog slides up 200ms ease-out |
| Form idle | Default input styles from Phase 1 |
| Validation error | `--destructive` border, error below field, 12px destructive text |
| Submitting | "Crear campana" button shows spinner, all inputs disabled |
| Success | Dialog closes, redirect to campaign detail, sonner toast |
| Error | Error message in destructive banner above form footer |

### Research Brief Editor

| State | Visual Treatment |
|-------|-----------------|
| Loading | 4 skeleton cards matching section layout |
| Empty (no content) | Each textarea shows placeholder helper text |
| Editing | Textarea focus: `--primary` focus ring, 2px offset |
| Unsaved changes | "Guardar" button pulses once (scale 1.02, 200ms) on first change, stays accent filled. Dot indicator next to "Guardar" text |
| Saving | "Guardar" button shows spinner, textareas remain editable (optimistic) |
| Save success | Sonner toast, dot indicator disappears |
| Save error | Sonner error toast, dot indicator remains |
| Critical path add | New row animates in with opacity + translateY, 200ms ease-out |
| Critical path delete | Row fades out 150ms ease, remaining rows shift up |

### Respondent Table

| State | Visual Treatment |
|-------|-----------------|
| Loading | 5 skeleton rows matching table layout |
| Empty | Centered empty state with icon and CTA |
| Row hover | `--surface-3` background, 150ms ease |
| Actions open | Dropdown menu appears below action button |
| Reminder sending | Row action button shows spinner |
| Reminder sent | Sonner toast, button returns to default |
| Delete confirmation | Alert Dialog with destructive styling |

### Voice Persona Selection

| State | Visual Treatment |
|-------|-----------------|
| Default | List of voice options, none selected initially (inherits campaign default if set) |
| Hover | Row background `--surface-2`, 150ms ease |
| Selected | Accent left border (3px), tinted background (primary 10% opacity) |
| Playing sample | Play icon becomes Pause icon (lucide Pause), accent color, audio plays |
| Sample ended | Pause returns to Play |

### Consent Screen

| State | Visual Treatment |
|-------|-----------------|
| Loading | Skeleton card centered on page |
| Default | All checkboxes unchecked, CTA button disabled |
| Partially checked | CTA remains disabled, checked items show accent checkmark |
